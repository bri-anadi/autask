# WEEK-11 Guide - Hardening dan Testing

## Scope Minggu 11
- Menstabilkan aplikasi untuk final demo.
- Menutup bug prioritas.
- Menjamin fallback manual saat AI gagal.
- Memperkuat coverage testing minimum.

## Langkah Pengerjaan Lengkap
1. Review skenario CRUD + AI + DB end-to-end.
2. Rapikan loading/empty/error state di UI.
3. Pastikan fallback manual selalu tersedia.
4. Tulis test case untuk alur penting.
5. Jalankan regression test dan catat bug sebelum/sesudah.
6. Tutup bug prioritas tinggi.

## Screenshot Wajib
- Screenshot state loading/empty/error final.
- Screenshot fallback manual saat AI gagal.
- Screenshot hasil test run.
- Screenshot bug log before/after.

## Penjelasan Wajib per Screenshot
- Perbaikan stabilitas yang dilakukan.
- Bukti fallback berjalan.
- Dampak hardening terhadap kesiapan release.

## Output Minggu 11
- LP9: test case, bug log sebelum/sesudah, ringkasan stabilitas.

## Format Pengumpulan
- Nama file: Kelompok_[Nomor_Kelompok].pdf
